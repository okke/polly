== 1. Overview

This document describes the design and implementation plan for a local-network
polling application.

The application allows a host to run a poll session on a laptop (MacBook) and
participants to join using their web browser via a URL shown on the host screen.
Participants must be connected to the same Wi-Fi network.

Participants respond to statements using a 4-point agreement scale.

The application is designed to be lightweight and easy to run locally.

Key design goals:

* No cloud dependencies
* Minimal infrastructure
* Simple configuration
* Realtime results
* Mobile friendly

The system uses:

* Vue.js frontend
* Ruby backend
* WebSockets for realtime updates
* JSON for poll definitions
* QR code for easy participant access

== 2. Functional Requirements

=== 2.1 Host Features

The host can:

* Start the poll server
* Load a poll from a JSON file
* Display the participant join URL
* Display a QR code for easy joining
* View live results
* Monitor the number of participants
* Export results

=== 2.2 Participant Features

Participants can:

* Join the poll using a browser
* Scan a QR code to access the poll
* View poll statements
* Select one answer per statement
* Change their vote if allowed
* Submit responses
* Receive confirmation

=== 2.3 Voting Scale

Each statement supports four answer options:

* Strongly Agree
* Agree
* Disagree
* Strongly Disagree

Internally these are represented as:

[source,json]
----
strongly_agree
agree
disagree
strongly_disagree
----

== 3. System Architecture

The system follows a simple client-server architecture.

[plantuml]
----
@startuml
node "Host Machine (MacBook)" {

  component "Ruby Poll Server" {
    component "REST API"
    component "WebSocket Server"
    component "Poll Loader"
    component "Vote Store"
  }

  component "Vue Frontend (built)"
}

node "Local WiFi Network"

node "Participants" {
  component "Mobile Browser"
  component "Laptop Browser"
}

Participants --> "Ruby Poll Server"
@enduml
----

Responsibilities:

Backend:

* Serve frontend assets
* Provide poll data
* Store votes
* Broadcast realtime updates

Frontend:

* Render poll UI
* Send votes
* Subscribe to realtime updates
* Render charts

== 8. API Design

=== 8.1 Get Poll

Request:

----
GET /api/poll
----

Response:

[source,json]
----
{
  "title": "Technology Opinions",
  "statements": [...]
}
----

=== 8.2 Submit Vote

Request:

----
POST /api/vote
----

[source,json]
----
{
  "participant_id": "uuid",
  "statement_id": "s1",
  "vote": "agree"
}
----

Response:

[source,json]
----
{
  "status": "ok"
}
----

=== 8.3 Get Results

----
GET /api/results
----

Returns aggregated vote counts.

=== 8.4 Export Results

----
GET /api/export
----

Returns CSV file containing all votes.

== 9. Realtime Updates

Realtime updates are required from the beginning of the project.

When a vote is submitted:

. Server stores vote
. Server recalculates results
. Server broadcasts update
. Admin dashboard updates automatically

Event flow:

[plantuml]
----
@startuml

participant Browser
participant Server
participant Dashboard

Browser -> Server : submit vote
Server -> Server : store vote
Server -> Dashboard : websocket update
Dashboard -> Dashboard : refresh charts

@enduml
----

WebSocket event:

[source,json]
----
{
  "type": "results_update",
  "data": {...}
}
----

Participants do not need live updates,
but the admin dashboard does.

== 12. Network Configuration (macOS)

To allow participants to connect to the host machine:

=== 12.1 Determine IP Address

Terminal command:

----
ipconfig getifaddr en0
----

Example output:

----
192.168.1.42
----

=== 12.2 Bind Server to Network

[source,ruby]
----
set :bind, "0.0.0.0"
----

=== 12.3 Firewall Settings

macOS Settings:

----
Privacy & Security
Firewall
Firewall Options
----

Allow:

* Ruby
* Terminal

=== 12.4 Server Startup

----
cd server
ruby app.rb
----

Participants connect using:

----
http://<host-ip>:4567
----

== 13. Duplicate Vote Prevention

Each participant receives a unique identifier.

Generated in the browser:

[source,javascript]
----
uuid = crypto.randomUUID()
localStorage.setItem("participantId", uuid)
----

The identifier is sent with every vote.

Server may enforce:

* One vote per statement per participant

== 15. Security Considerations

The application runs only on a local network.

Basic precautions:

* No external network exposure
* CORS restricted to local origin
* Random session ID optional

Example:

----
http://192.168.1.42:4567/poll/abc123
----
