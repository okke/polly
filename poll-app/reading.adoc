== Introduction

Recent advances in large language models and AI tooling have sparked an intense debate within the software engineering community about the future of programming.

Many researchers and practitioners argue that we are moving toward a new paradigm often described as *agentic software engineering*. In this paradigm:

* AI systems generate large portions of software
* autonomous agents perform development tasks
* developers supervise, guide, and validate outcomes rather than writing every line of code manually

This document provides an overview of several **influential essays, talks, and research papers** that have shaped the discussion around the future of programming and the emergence of agent-based software development.

The sources included here are widely referenced within engineering and AI communities.

---

== 1. Andrej Karpathy — Software 2.0

Author::
Andrej Karpathy

Link::
https://karpathy.medium.com/software-2-0-a64152b37c35

This influential essay introduced the concept of *Software 2.0*, a shift in which machine learning models replace traditional hand-written algorithms.

Karpathy describes two distinct programming paradigms:

[cols="1,3"]
|===
|Software 1.0
|Traditional software written explicitly by programmers.

|Software 2.0
|Software behavior learned through training neural networks on data.
|===

This essay laid the intellectual groundwork for later discussions about **AI-generated code and agentic programming systems**.

== 3. Simon Willison — Agentic Engineering Patterns

Author::
Simon Willison

Link::
https://simonwillison.net/2026/Feb/23/agentic-engineering-patterns/

This article explores practical patterns for working effectively with coding agents and AI-driven development workflows.

Willison argues that AI tools dramatically reduce the cost of generating code, which shifts the bottleneck toward validation and testing.

Important principles discussed include:

* iterative development loops
* strict testing discipline
* constrained execution environments for agents

== 5. Simon Willison — First Run the Tests

Author::
Simon Willison

Link::
https://simonwillison.net/guides/agentic-engineering-patterns/first-run-the-tests/

This article explains why automated tests are essential when working with AI coding agents.

Agents can rapidly generate and modify code, but without a robust testing framework, errors and regressions quickly accumulate.

The article emphasizes the importance of:

* comprehensive automated tests
* reproducible development environments
* strong feedback loops between agents and evaluation systems

== 9. Configuring Agentic AI Coding Tools

Authors::
Multiple researchers

Link::
https://arxiv.org/abs/2602.14690

This study analyzes thousands of open-source repositories that use AI coding agents.

The researchers discovered a growing pattern of configuration files that define how agents should operate within a repository.

One emerging convention is a configuration file called:

---

## AGENTS.md

Such files describe:

* agent permissions
* development workflows
* tool access
* evaluation strategies

This suggests that configuring AI agents may become a standard part of software engineering.

---

== 10. Why Agentic Pull Requests Get Rejected

Authors::
Multiple researchers

Link::
https://arxiv.org/abs/2602.04226

This research investigates pull requests generated by AI coding agents.

The study found that AI-generated pull requests are often rejected because of issues such as:

* incorrect assumptions about the codebase
* incomplete reasoning about system behavior
* improper tool usage

The findings highlight the continued importance of human review and engineering judgment.

---

== Conclusion

The essays and research papers described in this document illustrate a significant shift in how software is created.

Traditional programming workflows focus on developers writing code manually.

In contrast, emerging agentic workflows emphasize:

* describing goals
* orchestrating autonomous agents
* validating generated implementations
* supervising iterative development processes

Rather than eliminating programmers, these developments appear to **change the role of software engineers**, emphasizing architecture, system design, and evaluation of AI-generated solutions.

As AI systems continue to improve, understanding and mastering these new workflows will likely become an important skill for modern software engineers.
